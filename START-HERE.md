# НАЧНИ ОТСЮДА — без консоли

В архиве уже лежат две части:

- `plugin` — Figma-плагин;
- `server` — облачный Chromium, который открывает сайты по ссылке;
- `render.yaml` — готовые настройки для Render.

## Шаг 1. Загрузить проект на GitHub

1. Распакуй ZIP.
2. Открой созданный тобой репозиторий на GitHub.
3. Нажми **Add file → Upload files**.
4. Перетащи туда всё содержимое распакованной папки:
   - папку `plugin`;
   - папку `server`;
   - файл `render.yaml`;
   - остальные файлы из корня.
5. Внизу нажми **Commit changes**.

На главной странице репозитория должны быть видны папки `plugin` и `server`. Сам ZIP загружать не надо.

## Шаг 2. Запустить сервер на Render

1. Открой Render в браузере и войди через GitHub.
2. Разреши Render доступ к этому репозиторию.
3. В панели Render нажми **New → Blueprint**.
4. Выбери свой репозиторий.
5. Render найдёт файл `render.yaml` автоматически.
6. Нажми **Deploy Blueprint**.
7. Подожди, пока сервис `html-to-figma-render-server` получит статус **Live**.

Первый запуск может идти несколько минут: Render скачивает образ с Chromium.

## Шаг 3. Скопировать две вещи из Render

### Адрес сервера

На странице сервиса будет адрес примерно такого вида:

```text
https://html-to-figma-render-server.onrender.com
```

Скопируй его полностью.

### Ключ сервера

1. На странице сервиса открой вкладку **Environment**.
2. Найди переменную `API_KEY`.
3. Нажми показать/скопировать значение.

Это личный ключ. Не публикуй его в GitHub и не отправляй посторонним.

## Шаг 4. Подключить плагин в Figma

1. Открой **Figma Desktop**.
2. Открой любой дизайн-файл.
3. Перейди в **Plugins → Development → Import plugin from manifest…**.
4. Выбери файл:

```text
plugin/manifest.json
```

5. Запусти development-плагин.

## Шаг 5. Импортировать сайт

Во вкладке **Import from URL**:

1. В поле **Website URL** вставь ссылку на публичный сайт.
2. В поле **Render server address** вставь адрес Render.
3. В поле **Server key** вставь `API_KEY`.
4. Нажми **Check**.
5. Когда появится зелёная надпись о готовности, нажми **Import website**.

Адрес и ключ сохраняются в окне плагина, поэтому обычно их достаточно вставить один раз.

## Если бесплатный Render «спит»

После простоя первый запрос может запускаться дольше обычного. Нажми **Check**, подожди, пока сервер проснётся, и повтори проверку.

## Какие сайты проверять сначала

Начни с обычной публичной страницы без входа, CAPTCHA и личного кабинета. Сложные Canvas/WebGL-приложения, защищённые сайты и страницы с авторизацией могут импортироваться неполностью.
